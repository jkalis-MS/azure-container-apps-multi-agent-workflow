вҡҪ World Cup 2026 Match Simulator

A multi-agent system that simulates a Mexico vs Czechia World Cup 2026 match using AI-generated code running in isolated sandbox microVMs.

How It Works

Enter a prompt describing the match scenario. The system fires three agents in parallel:

рҹ”¬ Simulator Agent (Python / LangGraph) вҖ” Uses GPT-4o to generate Python code that searches the web for current team data, then generates more code to compute the simulation. All code executes in isolated sandboxes with an egress firewall.

рҹ“қ Blog Agent (.NET / Microsoft Agent Framework) вҖ” Takes the simulation result and generates a match report article using GPT-4o via IChatClient + ChatClientAgent.

рҹҺҷпёҸ Narration Agent (Python / GitHub Copilot SDK) вҖ” Generates a sports commentary script with multi-turn refinement via CopilotClient sessions, then converts it to audio using TTS-HD.

рҹ”’ Secure Egress вҖ” API Key Injection

When "Secure Egress" is enabled, the sandbox egress firewall uses a Transform rule to inject the Azure OpenAI API key into outgoing requests. The code inside the sandbox never has access to the real key.

WhatWithout Secure EgressWith Secure Egress вң…
/tmp/openai_request.txtShows real API key in headersShows INJECTED_BY_EGRESS_FIREWALL
Egress PolicyNo Transform ruleTransform: inject api-key from secret ref
Egress DecisionsвҖ”matchedRule: inject-aoai-key
Key storagePassed as env var to sandbox codeStored as group-level secret, resolved at egress

рҹ’Ў Demo tip: Open the simulation sandbox in the Portal вҶ’ browse /tmp/openai_request.txt or use the terminal to cat it. Compare both modes side by side.

Egress Firewall Demo

The sandbox egress policy blocks outbound traffic to specific domains. When AI-generated code tries to fetch from blocked sites, the connection is reset by the firewall вҖ” demonstrating how sandboxes protect against untrusted code making unauthorized external calls.
